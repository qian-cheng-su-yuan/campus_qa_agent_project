# 面试讲解稿

## 1. 一分钟介绍

这个项目是一个校园智能问答助手，主要解决学生查询校园办事流程不方便的问题。我把请假、宿舍报修、考试缓考、就业材料、图书馆和校园卡等资料整理成知识库，然后通过 RAG 的方式实现问答。用户提问后，系统先检索相关资料，再把资料和问题一起交给大模型生成回答，最后展示答案和参考片段。

## 2. 技术选型

后端使用 FastAPI，主要是因为接口开发简单、自动生成接口文档，适合快速做 AI 应用后端。前端用 Streamlit，方便本地演示。大模型调用采用 OpenAI-Compatible 接口，便于切换 Qwen、DeepSeek 或其他模型。

## 3. 核心流程

核心流程是：文档读取、文本切分、相似度检索、上下文拼接、Prompt 约束、大模型回答。这样做的好处是答案不是完全依赖模型记忆，而是尽量依据知识库内容，能降低幻觉风险。

## 4. 工程设计

项目按企业常见方式拆成 api、core、schemas、services、utils 几层。api 层只处理接口，services 层处理业务逻辑，core 层管理配置和日志，schemas 层定义请求和响应模型。这样后续扩展数据库、向量库或小程序端都比较方便。

## 5. 项目难点

主要难点是如何让模型不要乱编。我在 Prompt 中限制模型优先依据知识库回答，如果资料中没有明确依据，就要说明知识库没有找到明确说明。同时接口返回参考片段，便于用户检查答案来源。

## 6. 后续优化

后续可以把本地检索器替换成 Chroma、FAISS 或 Milvus，也可以增加后台管理页面，让学校老师维护知识库。如果部署到真实校园场景，还可以接入统一身份认证和微信小程序。
