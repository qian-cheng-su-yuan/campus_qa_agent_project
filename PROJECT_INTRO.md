# 项目介绍：校园智能问答助手

## 一、项目背景

在高校日常管理中，学生经常需要查询宿舍维修、请假审批、考试安排、就业材料、图书馆借阅、校园卡补办等信息。这些信息通常分散在学院通知、学生手册、QQ群公告、官网页面和辅导员通知中。学生遇到问题时往往需要反复询问老师或同学，查询效率不高。

本项目以校园服务咨询为场景，构建一个本地可演示的智能问答助手。系统通过整理校园常见办事资料，形成可检索的知识库。当用户输入问题后，系统先从知识库中检索相关片段，再将检索结果与用户问题一起发送给大模型，生成较为准确、结构化的回答。

## 二、项目目标

本项目主要实现以下目标：

1. 支持校园资料导入和知识库构建；
2. 支持学生用自然语言提问；
3. 能够返回办事流程、注意事项、所需材料等内容；
4. 回答时展示参考片段，便于确认答案来源；
5. 支持接入 API Key 调用真实大模型；
6. 支持无 API Key 情况下的本地演示；
7. 保持项目结构清晰，便于面试官查看代码。

## 三、系统架构

系统采用前后端分离但可本地一键演示的结构：

```text
Streamlit 前端页面
       ↓
FastAPI 后端接口
       ↓
RAGService 业务编排层
       ↓
DocumentLoader / TextSplitter / Retriever / LLMClient / AgentTools
       ↓
本地校园知识库 + 大模型 API
```

各模块职责如下：

- `web/streamlit_app.py`：负责页面展示、问题输入、结果展示；
- `app/api/v1/routes.py`：负责 API 路由；
- `app/services/rag_service.py`：负责 RAG 主流程编排；
- `app/services/document_loader.py`：负责文档读取；
- `app/services/text_splitter.py`：负责文本切分；
- `app/services/retriever.py`：负责相似度检索；
- `app/services/llm_client.py`：负责大模型接口调用；
- `app/services/agent_tools.py`：负责总结、关键词、办事清单等工具能力；
- `app/core/config.py`：负责读取环境变量配置。

## 四、核心流程

用户提问后，系统处理流程如下：

```text
1. 接收用户问题
2. 加载校园知识库文档
3. 对文档进行清洗和切分
4. 根据问题召回相关片段
5. 拼接 Prompt 和上下文
6. 调用大模型生成答案
7. 返回答案、参考片段、耗时和日志信息
```

## 五、Prompt 设计

系统 Prompt 的核心原则是：

- 优先依据知识库内容回答；
- 不确定的信息不能编造；
- 回答要适合学生理解；
- 办事类问题要按照步骤说明；
- 尽量给出材料、地点、时间和注意事项；
- 如果知识库没有依据，要明确提示“资料中未找到明确说明”。

## 六、适合面试展示的点

1. 不是单纯调用大模型，而是实现了完整 RAG 链路；
2. 代码分层清楚，能体现后端工程意识；
3. 支持 API Key 配置，能真实接入大模型；
4. 有本地兜底模式，避免演示时因为网络或余额问题失败；
5. 前端页面能直接展示结果，面试官能快速看到效果；
6. 参考片段可以说明答案来源，体现对大模型幻觉问题的处理。

## 七、后续优化方向

- 使用 Chroma / FAISS / Milvus 替换当前轻量检索器；
- 增加 PDF、Word、Excel 等更多文件解析；
- 增加管理员后台，用于维护校园知识库；
- 接入校园统一身份认证；
- 接入微信小程序或企业微信机器人；
- 增加反馈机制，让学生标记回答是否有用；
- 增加问答数据统计，分析高频咨询问题。
